# v5.0.1 - Bug Fix: Removed Mystery "0" Text

## ğŸ› Issue Fixed

The mysterious "0" text that appeared below every category on the dashboard is now gone!

## Root Cause

In the category display code, there was a `className="overflow-hidden"` on the outer div wrapper:

```tsx
// Before (v5.0.0)
<div key={cat.id} className="overflow-hidden">
  {/* category content */}
</div>
```

This was creating unwanted text nodes that displayed as "0" in the DOM. The `overflow-hidden` class was a leftover from when we were trying to fix spacing issues with inline styles.

## The Fix

Simply removed the `overflow-hidden` class:

```tsx
// After (v5.0.1)
<div key={cat.id}>
  {/* category content */}
</div>
```

## Why Did This Happen?

The "0" bug has been persistent across multiple versions:
- **v4.4.0-4.4.5**: Had `fontSize: '0'` inline style causing "0" to appear
- **v4.4.6**: Removed `fontSize: '0'` but left `overflow-hidden` class
- **v5.0.0**: Carried over the `overflow-hidden` class
- **v5.0.1**: Finally removed it completely! âœ…

The issue was that React/Next.js was creating a text node from the class name or some browser rendering quirk with the overflow property.
